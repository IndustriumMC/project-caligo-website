# Project Caligo landing page — baseline technical audit

Audited `index.html`, `styles.css`, and `script.js` against the rendered page at `http://127.0.0.1:4173/` on 2026-07-16.

## Audit health score

| # | Dimension | Score | Key finding |
|---|-----------|-------|-------------|
| 1 | Accessibility | 3/4 | Contrast and form semantics are strong; several navigation targets are below the 44px audit target. |
| 2 | Performance | 3/4 | The page is dependency-light, but the hero image renders at its 992px HTML height and both photographs ship only as JPEG. |
| 3 | Responsive Design | 2/4 | Breakpoints exist, but the oversized hero, one-column mobile outcome strip, and uneven mobile footer create avoidable vertical density. |
| 4 | Theming | 2/4 | Core tokens exist, but 32 hard-coded color/alpha usages bypass the token layer. |
| 5 | Anti-Patterns | 2/4 | The identity is distinct, but numbered process markers, mismatched decorative glyphs, and a collapsed synthetic butterfly mark read as generated scaffolding. |
| **Total** | | **12/20** | **Acceptable — significant work needed** |

## Anti-pattern verdict

**Partial fail.** The page does not look like a generic SaaS template: its typography, periwinkle palette, sharp borders, imagery, and commercial Minecraft positioning are distinctive. It still has visible AI-design tells: the `01 / 02 / 03` process markers, arbitrary `↗` and `◎` feature glyphs, repeated label-plus-oversized-heading compositions, and a CSS butterfly treatment that currently collapses into a cross. These details undermine an otherwise intentional identity.

## Executive summary

- Audit Health Score: **12/20 (Acceptable)**
- Issues: **0 P0 / 2 P1 / 5 P2 / 2 P3**
- Highest-impact problems: the 1,256px hero rendering bug, the 2px-tall Caligo wings, fixed-header hash targets, inconsistent decorative language, and mobile vertical excess.
- Recommended sequence: harden rendering first, then adapt responsive density, distill generated-looking decoration, and finish with polish.

## Detailed findings

### [P1] Hero image ignores the intended landscape ratio

- **Location:** `styles.css:344-350`, `index.html:62-68`
- **Category:** Responsive / Performance
- **Impact:** At 1280×720 the image renders at 550×992, making the hero 1,256px tall instead of a single decisive opening viewport. The primary value proposition and outcome strip no longer read as one composition.
- **Standard:** Responsive layout robustness; prevents avoidable below-the-fold displacement.
- **Recommendation:** Explicitly set `height: auto` and retain the declared `aspect-ratio`/`object-fit` treatment.
- **Suggested command:** `$impeccable harden`

### [P1] Caligo wing mark collapses to two horizontal pixels

- **Location:** `styles.css:772-800`
- **Category:** Responsive / Anti-Pattern
- **Impact:** Percentage height on flex children resolves to 2px in the rendered browser, leaving a cross where the identity mark should be. This looks broken in a brand-defining section.
- **Standard:** Responsive layout robustness.
- **Recommendation:** Give each wing a stable aspect ratio or clamped block size independent of unresolved percentage height.
- **Suggested command:** `$impeccable harden`

### [P2] Fixed header obscures hash navigation context

- **Location:** `styles.css:120-141`; all `main section[id]` anchors
- **Category:** Accessibility / Responsive
- **Impact:** Navigating directly to Benefits, Project Caligo, About, or Contact positions the section beneath the fixed 72px header; headings can appear clipped and users lose context.
- **WCAG/Standard:** WCAG 2.4.1/2.4.3 supporting navigation and focus order.
- **Recommendation:** Add a shared `scroll-margin-top` based on `--header-height`.
- **Suggested command:** `$impeccable adapt`

### [P2] Navigation and footer targets are undersized

- **Location:** `styles.css:143-206`, `styles.css:1038-1051`
- **Category:** Accessibility / Responsive
- **Impact:** Desktop navigation links measure 18px tall, the CTA and mobile toggle are 42px tall, and footer links are 16px tall. These are less forgiving for touch, motor impairment, and imprecise input.
- **WCAG/Standard:** WCAG 2.5.8 Target Size (Minimum); the audit target is 44×44px.
- **Recommendation:** Provide minimum block sizes and padding without enlarging the visible typography.
- **Suggested command:** `$impeccable adapt`

### [P2] Decorative language is inconsistent

- **Location:** `index.html:111-118`, `styles.css:512-516`
- **Category:** Anti-Pattern / Theming
- **Impact:** Serif arrow and bullseye glyphs feel unrelated to the mono infrastructure language used elsewhere, weakening visual cohesion.
- **Recommendation:** Replace arbitrary symbols with one consistent, tokenized feature-label motif.
- **Suggested command:** `$impeccable distill`

### [P2] Numbered process cards read as editorial scaffolding

- **Location:** `index.html:183-185`, `styles.css:723-761`
- **Category:** Anti-Pattern
- **Impact:** The detector flags `01 / 02 / 03`; the large empty gap beneath each number adds page length without improving comprehension. The ordered-list semantics already communicates sequence.
- **Recommendation:** Remove the decorative numbers, tighten the stage layout, and let the stage names carry hierarchy.
- **Suggested command:** `$impeccable distill`

### [P2] Mobile content density is unnecessarily tall

- **Location:** `styles.css:1430-1560`
- **Category:** Responsive
- **Impact:** Four outcome statements become four full-width rows, base sections retain 96px minimum vertical padding, and the final footer column sits alone. The page remains usable but feels longer and less composed on phones.
- **Recommendation:** Use a 2×2 outcome grid, a mobile section rhythm, and a deliberate full-width legal-link row.
- **Suggested command:** `$impeccable adapt`

### [P3] Theme tokens are incomplete

- **Location:** `styles.css` (32 literal color/alpha matches outside or alongside root tokens)
- **Category:** Theming
- **Impact:** Header glass, accent washes, nav text, action hover, and footer text use repeated literals. Future palette changes can drift and require search-and-replace.
- **Recommendation:** Promote repeated semantic colors and alpha surfaces to root tokens.
- **Suggested command:** `$impeccable colorize`

### [P3] Navigation lacks current-section feedback

- **Location:** `script.js:3-43`, `styles.css:164-195`
- **Category:** Accessibility / UX
- **Impact:** On a long landing page, the fixed navigation gives no indication of the section currently in view.
- **WCAG/Standard:** Supports WCAG 2.4.8 Location.
- **Recommendation:** Add a lightweight section observer and reflect state with `aria-current="location"` plus a restrained visual rule.
- **Suggested command:** `$impeccable onboard`

## Patterns and systemic issues

- Dimensions are fluid, but a few HTML presentational hints and percentage sizes are not explicitly normalized in CSS; both visible P1 defects come from that gap.
- The page has a strong token foundation, yet component-level alpha colors frequently bypass it.
- Spacing is consistently generous on desktop but not sufficiently recomposed for mobile; it mostly collapses columns without rethinking density.
- Repeated decorative shortcuts (symbols and numbers) are less considered than the imagery, typography, and copy.

## Positive findings

- All measured text tokens meet WCAG AA contrast on the midnight surfaces; primary and muted text generally approach or exceed AAA.
- Heading hierarchy, landmarks, image alternatives, persistent labels, required-field semantics, error text, and the live status region are well implemented.
- Keyboard focus is visible, Escape closes the mobile navigation, reduced-motion and reduced-transparency preferences are respected, and content remains usable without reveal gating.
- The page is lean: vanilla HTML/CSS/JS, no framework runtime, no layout-thrashing loops, lazy loading on the lower photograph, and only transform/opacity animation.
- The copy sells outcomes and support rather than exposing Mirage internals, matching the product register.

## Recommended actions

1. **[P1] `$impeccable harden`**: Normalize image and identity-mark geometry so the live page matches the authored layout.
2. **[P2] `$impeccable adapt`**: Fix anchor offsets, touch targets, mobile section rhythm, outcomes, and footer composition.
3. **[P2] `$impeccable distill`**: Remove numbered scaffolding and replace mismatched decorative glyphs.
4. **[P3] `$impeccable colorize`**: Consolidate repeated literal colors into semantic tokens.
5. **[P3] `$impeccable onboard`**: Add restrained current-section navigation feedback.
6. **[P3] `$impeccable polish`**: Re-check alignment, rhythm, hover/focus behavior, and optical details after the structural fixes.

Re-run `$impeccable audit` after fixes to measure the improvement.

# Character Rules

These rules apply to UI copy, Markdown prose, comments, documentation, commit messages, and placeholder content.

Use plain ASCII punctuation. The euro sign is also allowed.

Allowed punctuation:

`" ' - _ , ; : ! ? . / % *`

Use a hyphen only inside a compound word. Never use a spaced hyphen to separate clauses. Restructure the sentence instead.

Do not use smart quotes, curly apostrophes, em dashes, en dashes, ellipsis characters, arrows, copyright symbols, or other typographic punctuation.

Programming syntax is exempt when the language requires it.

Run `npm run check:characters` before committing. The production build runs the same check automatically.
